## Phase 1 — Survivors

### Injectable network seam
Problem solved: make every network call (HTTP/RPC/broadcast) substitutable so logic is unit-testable with zero network and zero funds.
Instances (3+):
- balance.py:64 — `fetch = fetch or _default_fetch` (EVM balance read)
- balance.py:39 — `(rpc or _default_solana_rpc)("getBalance", …)` (Solana balance)
- swap.py:125 — `(tx_build or _default_tx_build)(…)` (Paraswap tx build)
- gas.py:59/63 — `(quote_fetch or _default_quote_fetch)` / `(submit or _default_submit)` (gasless)
Reference instance: balance.py (cleanest, most-recent 2026-06-01, most-tested).
Canonical shape: the function takes an optional callable param defaulting to None (balance.py:51); inside it resolves `dep = dep or _default_dep` (balance.py:64); the default does the real network call, tests pass a fake that returns canned data and/or captures the request (balance.py:39).
Known trap: forgetting to ALSO validate the response — get_balance never checks Etherscan's `status` field, so an API error payload raises an opaque error on `int()` (logged in STATUS.md "Known limitations to harden later").
Test contract: tests/test_solana_balance.py — injects `rpc=lambda m,p:{"value":…}` and asserts the converted value + that the right method/address was queried.
When to use: any function that touches an external network/RPC/HTTP service and needs deterministic tests.
When NOT to use: pure-compute functions with no I/O (adds a useless param).

### Chain-dispatch guard
Problem solved: route one public API across multiple chains while rejecting unknown chains with a clear, consistent error.
Instances (3+):
- wallet.py:24-26 + wallet.py:131 — create_wallet dispatch (evm/solana/bitcoin else ValueError)
- balance.py:56-58 — get_balance dispatch
- tx.py:46-48 — send guard; swap.py:34-35 — swap guard; gas.py:46-47 — gasless guard
Reference instance: wallet.py:118-131 (create_wallet) — cleanest explicit dispatch, last-touched 2026-06-01.
Canonical shape: a module-level `SUPPORTED_CHAINS` tuple (wallet.py:9); the entry function branches `if chain == "<x>": return <x_impl>(…)` per chain (wallet.py:119-130); a final `raise ValueError(f"unsupported chain {chain!r}; supported: …")` (wallet.py:131).
Known trap: a chain added to `SUPPORTED_CHAINS` but NOT given a routing branch silently falls through to the wrong adapter or a confusing error — EVM `Wallet` stays evm-only on purpose; routing lives in create_wallet, not in the class (wallet.py commit d4f6cc2).
Test contract: tests/test_wallet.py::test_unsupported_chain_raises_value_error and the per-chain create_wallet tests.
When to use: any multi-chain/multi-backend entry point.
When NOT to use: single-backend code (the tuple + branch is overhead).

### Sign-and-broadcast recipe (EVM)
Problem solved: build, locally sign with the owner's key, and optionally broadcast an EIP-1559 tx — non-custodial, broadcast injectable.
Instances (3+):
- tx.py:60-66 — send (native transfer)
- tx.py:108-121 — call_contract
- swap.py:71-87 — swap (0x); swap.py:131-145 — swap_paraswap
Reference instance: tx.py:54-66 (send) — simplest, last-touched 2026-05-31.
Canonical shape: build the EIP-1559 dict (`type:2`, nonce/gas/fees/chainId) (tx.py:50-59); `signed = Account.sign_transaction(tx, wallet.private_key)` then `raw_hex = _to_0x_hex(signed.raw_transaction)` (tx.py:60-62); `if broadcast is not None: returned = broadcast(raw_hex); if returned: tx_hash = returned` (tx.py:63-65).
Known trap: the locally-computed tx hash EQUALS the network hash, so a returned hash does NOT confirm the broadcast succeeded — a real mainnet send sat pending and a replacement mined with status 0x0 (revert) before a clean send; always confirm via receipt (STATUS.md "LIVE PROOF" + commit 9cd2092).
Test contract: tests/test_send.py::test_signed_transaction_recovers_to_the_owner_address (recovers signer from raw).
When to use: any EVM state-changing transaction.
When NOT to use: non-EVM chains (Solana/Bitcoin use the same SHAPE but different libs — do not force this exact code).

### Key-secrecy wallet invariant
Problem solved: hold a private key the owner can use to sign, while guaranteeing it never leaks into representations, logs, or shared info.
Instances (3+):
- wallet.py:31,43-53 — EVM Wallet
- wallet.py:63,75-84 — SolanaWallet
- wallet.py:95,107-116 — BitcoinWallet
Reference instance: EVM Wallet (wallet.py:22-53) — first and most-tested.
Canonical shape: store the secret as `self._private_key` (wallet.py:31); expose it only via a `.private_key` property used solely for signing (wallet.py:43-44); `public_info()` returns `{address, chain}` only and `__repr__`/`__str__` print address only, with `__str__ = __repr__` (wallet.py:46-53).
Known trap: the key still lives in the instance `__dict__`, so `vars()`/`pickle` would expose it — no redaction test yet (STATUS.md "Known limitations to harden later"; flagged by the green-review reviewer).
Test contract: tests/test_wallet.py::test_private_key_never_leaks_into_repr (+ solana/bitcoin equivalents).
When to use: any object that holds a secret the owner must use but must not leak.
When NOT to use: objects with no secret to protect.
